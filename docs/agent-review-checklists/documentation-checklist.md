# Documentation Review Checklist

Use this checklist to review business-facing documentation.

- [x] README explains what the project is.
- [x] README explains what problem it solves.
- [x] README includes architecture overview.
- [x] README includes prerequisites.
- [x] README includes local development commands.
- [x] README includes Azure deployment with azd.
- [x] README includes demo flow.
- [x] README includes KPI examples.
- [x] README includes next steps.
- [x] Architecture document explains services in business terms.
- [x] Stakeholder discovery template exists.
- [x] SOP AI intake process exists.
- [x] KPI scorecard exists.
- [x] Vendor evaluation template exists.
- [x] Demo script exists.
- [x] End-user consumption document exists for SharePoint, Microsoft 365, and Hermes patterns.
- [x] Agent skill registry document exists for vector skill lookup and department/task grouping.

## Follow-Up Checks

- [ ] Ask a non-technical reviewer whether the architecture explanation is clear.
- [ ] Replace TODO KPI targets after stakeholder discovery.
- [ ] Add screenshots after the first successful deployment.
