# Demo Readiness Checklist

Use this checklist before an interview or stakeholder demo.

- [x] README gives a clear demo path.
- [x] Demo script exists.
- [x] Architecture document supports a business-facing walkthrough.
- [x] API has simple endpoints that can be called locally.
- [x] KPI scorecard shows how success will be measured.
- [x] Placeholders are explicit instead of hidden.

## Follow-Up Checks

- [ ] Run the API locally before the demo.
- [ ] Prepare one sample document for `/upload`.
- [ ] Prepare one business question for `/ask`.
- [ ] Decide whether to demo local-only or deployed Azure flow.
- [ ] Review Azure costs before running `azd up`.
