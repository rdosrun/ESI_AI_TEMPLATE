# Project Instructions

This repo is an Azure AI Solutions Architect starter kit.

Priorities:
- Keep the project interview-demo friendly.
- Use Azure Developer CLI and Bicep.
- Prefer simple, readable infrastructure modules.
- Do not hardcode secrets.
- Do not commit .azure/, .env files, or .codex/.
- Explain technical choices in business terms.
- Keep documentation useful for both technical and non-technical readers.
- Use Python FastAPI for the starter API.
- Use Azure Container Apps as the default hosting target.
- Use TODO comments where real Azure subscription/model details are required.

Before major changes:
- Inspect existing files.
- Avoid overwriting useful work.
- Summarize planned changes.

After changes:
- Summarize files changed.
- Show commands to run next.
- List assumptions and TODOs.
